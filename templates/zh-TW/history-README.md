# 歷史

這個目錄放**因果軌跡**——為什麼專案變成現在這樣。不是流水帳事件，而是**決策的轉移**與**被否決的選項**。它們是蒸餾成 experience.md 的原始素材。

> 為何叫 `history` 不叫 `migrations` / `decisions`（被否決選項的墓碑）：`migrations` 只指**結構變更**、且**軟體特定**；`decisions` 丟掉「轉移／superseded」那層時間性。`history` 最通用——涵蓋決策轉移、否決選項、系統自身演化，且任何領域都通。

## 什麼時候在這裡新增檔案
- 一個決定**取代**了先前的決定——記下舊的、新的，以及**為什麼變**；舊的標 superseded 並連到新的。
- 一個選項**被否決**了——留**墓碑**：考慮過什麼、**為什麼不選**（最濃的 why）。
- 一次除錯或里程碑揭示了重要的轉折。

## 建議格式

```markdown
# [NNN]：從 [舊] 到 [新]
> 日期：YYYY-MM-DD

## 轉移
- 舊：...
- 新：...

## 為什麼變
...

## 狀態
✅ 已採用  /  ⚰️ 已否決（理由）
```

## 檔案命名與日期
用編號前綴保持因果順序：`001-...`、`002-...`（NNN 是主鍵，別把日期塞進檔名——會變「編號 + 時間」兩個軸混在一起）。
日期放 body（`> 日期：YYYY-MM-DD`），標到**天**即可：時間是過度精度，NNN 已經把同一天的多筆轉移排好順序了。（日期讓「根本原則改動率」這類隨時間算的健康指標有得算。）

## 與核心文件的關係
這裡是未壓縮的因果軌跡；其中的教訓蒸餾進 **experience.md**（四段式）。一條反覆出現的軌跡成熟後，蒸餾成 experience 的一條教訓。

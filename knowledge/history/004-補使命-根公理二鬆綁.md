# 004：補上使命「為共識而記」、根公理二鬆綁為「記憶可大致回溯」

> 一條**根本原則修訂**的因果記錄（knowie 第一次動根本原則，走特殊路徑）。

## 轉移
- **新增 使命（telos）**：為共識而記——knowie 為「人＋AI 的共識」而記憶。
- **根公理二**：「knowie 管 why，不管 what」→ **superseded** → 「記憶可（大致）回溯」（記 why + 回溯所需最小 how/what；不複製 code 的 what，但不是零 what）。
- **可回憶**：不另列——它就是新根公理二的範圍判準。

## 為什麼變
1. **填洞**：延伸原則 3/4/5/7 一直引用「使命」，但使命從未列為根本原則（principles.md 自己標的自洽缺口）。現有根本原則全是 🟡（未經人定案）。所以這是補洞 + 定案，不是推翻穩定根。
2. **為何「為共識而記」當 telos（不是「可回憶」）**：可回憶是機制、agent-only 也能做；為共識而記回答「為誰記」（人＋AI 共識），推導力更強——人在環、superseded 留痕、衝突呈現給人、結構化人類可讀，這些可回憶單獨推不出、為共識乾淨推出。
3. **為何根公理二鬆綁**：使用者指出「管 why 不管 what」太硬。舊表述「完全不管 what」與「可回憶單元＝why+how+概念」衝突——要回溯就得留一點 how/what。鬆綁為「回溯所需」後一致。

## 被否決 / 取代
- **可回憶當根公理（與為共識並列）**：降為「根公理二的判準」，不並列。它是手段，為共識是目的，分兩層。
- **「為 agent 而記」（Letta / Hermes 的根）**：knowie 明確不選這條——它推出全自動、貪婪、覆寫；knowie 走人在環、保守、留痕。兩個根各自的設計都是各自乾淨投影。
- **舊根公理二「管 why，不管 what」**：superseded，不刪——保留為「回溯所需」這條的前身，舊的精神（不複製 code 的 what）併入新表述下半句。

## 旁證（多條獨立線指向同一根）
競品「為誰而記」對比、連結/檢索收斂（「檢索是手段、共識是目的」見 [003](003-連結慣例與檢索定位.md)）、競品修正（knowie 獨特的是「結構化的 why」）——三條獨立推理都落在「為共識而記」。多線指同一根 = 通常是真的根。

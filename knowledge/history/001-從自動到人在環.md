# 001：從「AI 自動分類」到「人在環定案」

> 一條設計轉移的因果記錄。

## 轉移
- **A（曾考慮）**：AI 動態分類、全自動維護知識庫。
- **B（現採用）**：AI 做可逆/機械的；定案、修訂、認定母概念由人。

## 為什麼變
最初對「AI 自動分類」踩煞車，理由是「位置要對人的肌肉記憶可預測」。但這個前提**錯了**——若 knowie 的讀者主要是 AI（每次讀索引導航），「位置漂移害人找不到」就不成立。

修正後的真正理由不是「AI 不該自動」，而是：**LLM 發散、會漂移**，所以自我維護必須套「收斂軌道 + 可審計」。可逆操作可全自動（git 留痕、人事後審）；不可逆的語義固化（定案/修訂）才必須人在環，因為它的錯誤不可逆且傷及權威層。

## 旁證
2026 agent memory 主流（Letta 等）走全自動 agent 自我編輯，Letta 自曝「risky—可能誤刪重要記憶」——正是這條轉移要避開的坑。knowie 的「定案不可外包」是這個坑的解。

## 狀態
✅ 已採用（B）。延伸原則 5「可逆的自動化，不可逆的要人」即此轉移的結晶。

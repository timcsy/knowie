# 007：`knowie-crystallize` → `knowie-consolidate`（固化）

> 2026-06-11。一條命名轉移：skill 改名，行為不變。

## 轉移
- 舊：`knowie-crystallize`（中文描述用「結晶」）。
- 新：`knowie-consolidate`（中文用「固化」）。skill 行為完全不變（人主動把成熟 draft 分發到長期層）。

## 為什麼變
1. **撞車**：精煉這條隱喻的詞（`結晶`、`蒸餾`）已散在三處——根本原則 2「概念從具體中**結晶**」（概念湧現）、[蒸餾](../concepts/蒸餾.md)（精煉概念）、[記憶系統](../concepts/記憶系統.md)「三視角是**結晶**介面」。再用 crystallize 當 skill 名，會把這些攪在一起。
2. **更準**：這個 skill 做的是「工作記憶（draft）→ 長期記憶」的搬遷，認知科學標準詞就是 **consolidation（固化/鞏固）**——不是隱喻。
3. **補完正典三階段**：[記憶系統](../concepts/記憶系統.md) 整套鋪在認知記憶上，skill 三件套因此對齊記憶形成：**capture（編碼）→ consolidate（固化）→ next（提取）**。crystallize 對不上這條軸，consolidate 對得上。
4. **消歧**：改名後，`結晶` 只留給「概念湧現／結晶介面」、`蒸餾` 只留給精煉概念、`consolidate/固化` 只指這個 skill——三詞各歸各位。

## 被否決的選項（純金 why）
- **`distill`（蒸餾）**：已被概念佔用；且 distill 是「精煉（品質軸）」，不是「in→長期（位置軸）」——不同軸，別混。
- **`promote` / `commit` / `file`**：太軟體味、無記憶語意。
- **`沉澱`（settle）**：太被動（這是人**主動**發起的）；且 [蒸餾](../concepts/蒸餾.md) 已明示避開「沉澱」這類「暗示已保留」的詞。

## 一個 caveat（已在 skill/概念裡框好）
`consolidate` 日常義偏「合併成一」，但本 skill 是**分發到多層**（[分發非傾倒](../concepts/分發非傾倒.md)）。框法：記憶科學的 consolidation 本就不是「合併成一」，而是「不穩→穩定進長期庫」——**knowie 的固化靠分發、不靠 dump**。

# 已定案的路線發現是錯的，怎麼辦（對抗認知失調）

> 2026-06-06 設計討論。draft，未定案。
> 怎麼冒出來的：問「draft 結晶到三檔、據以開發一陣子後，突然發現整條路線錯了」怎麼處理——這是 knowie 的真正考驗，呼應「怎麼知道 knowie 好不好」裡的失敗模式。

## 問題的本質
不是「想法漸進過時（superseded）」，而是「**已定案 + 已據以行動的東西被發現根本錯**」= 認知失調。最危險，因為沉沒成本 + 確認偏誤會誘使**合理化維持**，而不是認錯（這正是 knowie 一貫的敵人：自我服務的漂移）。

## 核心回答：讓「認錯」變便宜
死撐的最大藉口是「認錯成本太高」。knowie 逐項降低它，於是合理化的誘惑變小：
- **因果軌跡**：認錯 = 記一條轉移（舊的標 superseded + 為什麼錯），不是摧毀。
- **superseded 不刪**：錯誤留著當教訓——最寶貴的 why（「為什麼不能走這條」）。
- **推導鏈**：知道錯根牽連哪些衍生，連帶更新可追蹤。
- **可回憶單元**：回得到「當初為何選這條」，精確診斷，不整團丟棄。

## 處理流程
1. **承認 + 標記（🔴），不立刻刪、不合理化。** 先擋住「捏理由維持」。
2. **解剖：不是整條錯，是哪個根前提錯。** 根公理反過來用：一個根錯 → 投影全歪。用可回憶單元找那個根，把大危機縮小成「一個根的問題」。
3. **修訂根本原則**（若錯的是原則/根公理）：現實反饋有資格挑戰原則；標 superseded、記因果、新的取代舊的。
4. **沿推導鏈連帶更新**：錯根的所有投影（衍生原則 / vision / code）一起改。延伸原則標「推導自哪條」，就是為了出事時能追蹤牽連範圍。
5. **蒸餾「為什麼會選錯」成 experience**：錯誤路線 → 一條防重蹈的 meta 教訓（錯誤本身變成 why）。

## 限制
- knowie **不替你發現錯**：方向性的錯要靠人對現實的洞察；AI 只能偵測局部 🔴 不一致。knowie 給的是「認錯後有序 + 認錯便宜」。
- 最深的認知失調（**根公理本身錯**）仍最痛；但它可標記、可修訂、連帶影響沿推導鏈可追蹤——痛有出口，不是茫然推翻一切。

## 為什麼這條重要
**knowie 會不會爛，很大程度取決於它能不能對抗認知失調**——讓人和 AI 願意認「已定案的錯」。這是失敗模式的核心要害。

# 根公理候選：為共識而記 vs 可回憶

> 2026-06-06。draft，未定案。**對現有根本原則的重大修訂提案。**
> 怎麼冒出來的：從競品「為誰而記」的對比，浮現「為共識而記」這個候選；它跟「記憶系統框架」那條的「可回憶」候選打對台。

## 現有的根公理（principles）
1. 一個概念，多個投影。
2. 管 why，不管 what。

## 兩個新候選
- **A. 可回憶**：讓專案過去的認知能被完整重建（為何 / 怎麼 / 用了哪些概念）。
- **B. 為共識而記**：knowie 為「人 + AI 的共識」而記憶——讓雙方對齊、人保有定案主權、可回溯為什麼。

## 關係：B 是目的，A 是手段
- 可回憶（A）是為了**共識（B）能被重建**。
- 管 why、一個概念多投影、人在環——也都是達成 B 的方法。
- 所以 **B 似乎最根本**：其餘（①②、A）都是它的投影 / 手段。

## B 能推出 knowie 幾乎所有設計（它的投影）
從「為共識而記」乾淨推出：
- 保守准入 + 人在環定案（共識的錯誤後果重、主權在人）
- 遺忘留痕（墓碑 / superseded：共識有歷史維度）
- 結構化視角 + 人類可讀（人要讀 / 審）
- 法典分層（共識要穩定錨）
- `history`（共識是「我們怎麼到這裡」）
- 衝突呈現給人（共識衝突要裁決，非自動消）
- 社會性（給第三方、跨專案：共識面向多人 / 跨時）
- ROI = 決策變好 / 共識變清（非 agent 變強）

反過來「**為 agent 而記**」（Letta / Hermes）推出：全自動、貪婪、覆寫、功能分層。**兩個根，各自的設計都是各自的乾淨投影。**

- 「可回憶」（A）是降為 B 的手段，還是與 B 並列？

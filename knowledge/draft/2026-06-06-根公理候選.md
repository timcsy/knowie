# 根公理候選：為共識而記 vs 可回憶

> 2026-06-06。draft，未定案。**對現有根本原則的重大修訂提案。**
> 怎麼冒出來的：從競品「為誰而記」的對比，浮現「為共識而記」這個候選；它跟「記憶系統框架」那條的「可回憶」候選打對台。

## 現有的根公理（principles）
1. 一個概念，多個投影。
2. 管 why，不管 what。

## 兩個新候選
- **A. 可回憶**：讓專案過去的認知能被完整重建（為何 / 怎麼 / 用了哪些概念）。
- **B. 為共識而記**：knowie 為「人 + AI 的共識」而記憶——讓雙方對齊、人保有定案主權、可回溯為什麼。

## 關係：B 是目的，A 是手段
- 可回憶（A）是為了**共識（B）能被重建**。
- 管 why、一個概念多投影、人在環——也都是達成 B 的方法。
- 所以 **B 似乎最根本**：其餘（①②、A）都是它的投影 / 手段。

## B 能推出 knowie 幾乎所有設計（它的投影）
從「為共識而記」乾淨推出：
- 保守准入 + 人在環定案（共識的錯誤後果重、主權在人）
- 遺忘留痕（墓碑 / superseded：共識有歷史維度）
- 結構化視角 + 人類可讀（人要讀 / 審）
- 法典分層（共識要穩定錨）
- `history`（共識是「我們怎麼到這裡」）
- 衝突呈現給人（共識衝突要裁決，非自動消）
- 社會性（給第三方、跨專案：共識面向多人 / 跨時）
- ROI = 決策變好 / 共識變清（非 agent 變強）

反過來「**為 agent 而記**」（Letta / Hermes）推出：全自動、貪婪、覆寫、功能分層。**兩個根，各自的設計都是各自的乾淨投影。**

## 收斂（2026-06-07）：B 當使命（telos），①② 是方法層，A 降為手段
多條獨立推理在這次都指向 B（為共識而記）：競品「為誰而記」對比、連結/檢索收斂（「檢索是手段、共識是目的」見 [history/003](../history/003-連結慣例與檢索定位.md)）、競品修正（knowie 獨特的是「結構化的 why」不是人在環本身）。多條獨立線指同一根 = 通常是真的根。

**Q1：B 統攝還是並存？→ 兩層,不是統攝。**
- **使命層（telos）**：B（為共識而記）——回答「為何記、為誰記」。
- **方法層（公理）**：①一個概念多投影（知識論方法，源自 Frame，承重牆是經驗，**半獨立**——不硬塞成從 B 推導）；②管 why 不管 what（範圍，**部分可從 B 推**:共識關乎 why）。
- 不讓 ① 假裝「推導自 B」(那是 fake 推導);讓它作為服務 telos 的方法公理並存。

**Q2：A（可回憶）降為手段。**
- A 是 B 的手段:共識要能被重建 → 過去認知可回憶。→ A 變**延伸原則**（推導自使命），不是根公理。

**為何 B 勝 A 當 telos**：A（可回憶）是機制、能 agent-only;B 答「為誰（人＋AI 共識）」,且推導力更強——人在環、superseded 留痕、衝突呈現給人、結構化人類可讀,這些 A 單獨推不出,B 乾淨推出。

**為何此修訂風險低（不算 churn）**：principles.md 早有自洽缺口——延伸原則 3/4/5/7 都引用「使命」,但使命從未列為根本原則;現有根本原則全標 🟡（未經人定案）。所以這是**填一個已知的洞 + 定案 🟡**,不是推翻穩定的根。

## 提案的 principles 根本原則結構（待人拍板才動）
```
根本原則
  使命：為共識而記（人＋AI 對齊、人保定案主權、可回溯為什麼）   ← 新增,填洞
  根公理一：一個概念，多個投影（方法,半獨立）
  根公理二：管 why，不管 what（範圍,部分推導自使命）
延伸原則
  + 可回憶：推導自使命（共識要能被重建）                       ← A 降此層
```
- 走修訂流程:記因果（history）、舊狀態（🟡 缺口）標說明、新的取代。
- ⚠️ **動根本原則是不可逆的人類判斷**（延伸原則 5）——AI 只提案到這,等人拍板。

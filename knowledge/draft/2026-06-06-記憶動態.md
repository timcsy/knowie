# 記憶的動態：寫入、整理、回流、對齊

> 2026-06-06。draft，未定案。
> **吸收並取代**：`draft整理機制`、`設計型draft無法預先測`，以及這幾輪討論的認識鏈 / 設計型出口 / experience 來源 / episodes 對齊 / vision 整理。
> 與「記憶系統框架」互補：那條是靜態解剖（有哪些記憶層），這條是動態生理（東西怎麼流動）。

## 兩條入口線：想 vs 做
- **想 → draft**：未定案的思考（問題 / 設計 / 討論洞察）。
- **做 → 行動回流**：實作完成的經驗（**不經 draft**）。
- ⚠️ experience 的**主來源是「做」**，不是 draft（曾搞錯）。

## draft 內：認識成熟鏈（三型，遞進）
問題型（未知）→ 設計型（構想未驗）→ 經驗型（已驗證）。各有**不同出口**：
- **問題型** → 找答案。
- **設計型** → （人定案）進 vision roadmap（前瞻）→ 交棒實作 → **draft 放手**。
- **討論洞察（想出的經驗）** → 固化 concepts / principles。

## 複習信號 = 推進一階（不是直接固化）
複習（同主題反覆）= 「該往前一階」，往哪依當前階段：經驗型→固化；設計型→進 roadmap；問題型→找方案。

## 行動回流：做完 roadmap → 三目的地 + 出列
- 教訓（蒸餾）→ **experience**
- 完整經歷（值得回憶）→ **episodes**（現場 / 可回憶單元）
- 決策轉移（若有）→ **history**
- **roadmap 出列**（兌現後不留 vision）

## experience ↔ episodes 對齊：靠連結
- experience = 精華索引（教訓 + 指向 episode）；episodes = 現場。
- 兩來源（A：做完即時分流 / B：experience 太滿事後下沉）= **同一分流的兩個時機**，靠「教訓 ↔ episode 連結」對齊，同次經歷不重複。理想是 A，B 是補救。

## 各記憶層的整理（越用越健康）
- **concepts**：向上收斂（找母概念）。
- **experience**：雙向（向上收斂母教訓 + 向下分離現場→episodes + 清過時）。
- **vision**：兌現出列（做完→experience/episodes/history，roadmap 清掉）——**時間清理，非抽象蒸餾**；願望膨脹→沒承諾的回 draft（roadmap 只放承諾的）。
- **draft**：衰減（久沒碰淡出）/ 固化（複習）。
- **episodes**：入口篩選（只存值得回憶的，其餘讓現場淡去）。

## 設計型 draft 的驗證（雞生蛋）
設計型要實作才知有效、實作有成本。破解：**dogfood**（元層，如 judge 試跑）/ **真實專案**（領域層）/ **內省 vs 實測** 分流（能想清楚的就定，行為·規模·人因的設最小實驗）/ **借既有**（Letta·Hermes·agentskills.io，別自己造來測）。

## judge 試跑暴露的缺口（待硬化）
孤兒 / 死連結 / recency 可機械偵測；**複習信號（同主題反覆）還沒機械化，靠 AI 讀**——judge 的軟介面。硬化方向（FUSE 式）：定義「主題 tag 結構 + 偵測協議」，底層相似度可插拔（grep / 嵌入），knowie 不自己寫演算法。規模到了再做。

# 競品三方定位（vs Letta / Hermes）與生存空間

> 2026-06-06，查證 Letta + Nous Research 的 Hermes Agent 後的對比。draft，未定案。
> 怎麼冒出來的：問「knowie 跟 Letta / Hermes Agent 距離多遠」，查完發現 knowie 的「記憶 agent」野心大半已被它們 production 覆蓋，但有一塊它們刻意留白。

## 三方是什麼
- **Letta**（MemGPT 系）：OS 式記憶平台，agent 自治、git-backed MemFS、skill learning。Production。
- **Hermes Agent**（Nous Research，2026/02，v0.9）：自主 agent + 多層記憶（MEMORY.md/USER.md 常駐 + SQLite FTS5 session search + 外部 provider）+ 自習得 skill（agentskills.io 開放標準）+ 全自動 + 多平台。Production。
- **knowie**：why 的協議，人在環、結構化（三視角/法典）、寄生 markdown。理論深、實作初級。

## 三方共通（已是業界共識）
markdown 記憶、多層、skill = 程序記憶、容量逼 curation、explicit + implicit 存、FTS5 + LLM 雙檢索。**這些 knowie 講的，它們都 production 了。**

## knowie 的距離
- **工程 / 能跑**：落後很遠（它們 production；knowie 協議 + 剛重寫未驗證 skill + draft 理論）。
- **程序自習得**：**Hermes 領先**（已做 + 開放標準化；knowie 還在 draft）。
- **可回憶 / 檢索**：概念打平、實作落後（它們有 FTS5 + LLM；knowie 的「可回憶單元」更深但紙上）。
- **結構化 why + 人在環**：**knowie 獨特**——兩者都沒 why 層（Hermes：why「distilled into facts if critical」）、都全自動無確認。

## 最尖銳的結論
knowie 的「記憶系統」野心，在「facts + skills + 全自動」版本上已被輾過（Hermes 連程序自習得都做完標準化了）。**唯一沒被覆蓋的，是它們刻意不做的兩件事：結構化 why（三視角 / 法典 / 因果）+ 人在環定案（captured ≠ committed）。**

## 戰略（生存空間）
1. **別比「記憶 agent」**——會被輾壓。
2. **當它們的「why 層」**——寄生在 Hermes / Letta / 任何 agent 之上，提供它們缺的「結構化 why + 人的定案主權」。這是「協議寄生平台」的終局。
3. **程序記憶別自己造**——用 agentskills.io 開放標準（Hermes 已採用）；knowie 專注 why。
4. **定位轉變**：knowie 不是「又一個記憶 agent」，是「facts/skills 之上、它們留白的 **why 層 / 人在環定案層**」。越早這樣定位，生存空間越清楚。

## 對 vision 的影響（待落地）
這直接影響 knowie 的定位，將來可能該寫進 vision：**knowie = 寄生於 agent 記憶系統之上的 why 協議，不與記憶 agent 競爭**。現在留 draft，未定案。

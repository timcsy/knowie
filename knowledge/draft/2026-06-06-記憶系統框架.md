## 程序記憶：自習得
讓 LLM 把常做的事自己蒸餾成 skill。

- **兩種來源**：meta skill（維護知識庫：capture/judge）+ **domain skill**（專案領域的重複工作：爬蟲建題庫、批量轉譯…）。domain 更有價值——幫專案長出「領域能力庫」。
- **= capture 的程序版**：察覺操作重複 → 記一條「候選 skill」進 draft（連這次怎麼做的）→ 反覆／成熟 →（人確認）**固化進 `knowledge/skills/`（小腦）→ 安裝到 runtime 變可執行**。不需新引擎、不需操作日誌（靠在場察覺，把過眼即忘的 transcript 搶救成 draft）。
- **邊界**：做一次 → spec 工具實作；反覆做 → 固化成 domain skill（複習效應）。
- **守紀律**：skill 化是定案級、且影響更大（被執行，非被引用）→ AI 提議、人確認，比知識定案更慎。
- **skill 哲學分叉**：knowie 的「判準不窮舉」是給 **meta skill**（導引推理，情境無窮）；**domain skill** 自動化的是**機械重複工作** → **可/該更程序化（rote 步驟）**，別硬塞判準式。（仍是光譜：domain 遇變異也要點判斷。）
- **整合非自造**：採 agentskills.io 開放標準（Hermes 已採），別造引擎。
- **理論基礎（2026-06-07）**：**skill 是知識**（對 LLM：能力＝context 裡的資料，code＝data）。所以「重用 capture / 記憶動態」不是巧合——程序層跟其他層**同物質、同動態**，只是 stakes 較高（執行級 → 人確認更嚴）。連帶：固化 domain skill 要記 **why-behind-skill**（寫在 skill 裡）。⚠️ 修正：skill 體**存進小腦、不指出去**（它是 knowie 自己的記憶，非外部物；只有 code/spec 才指出去）。見 [concepts/記憶系統](../concepts/記憶系統.md)。

## 固化 + 同步（2026-06-07 查證 agentskills.io / Claude / Codex / VS Code）
- **格式抄 agentskills.io**（開放標準，Codex/VS Code/Claude 都採；Hermes 也用）：一資料夾 + `SKILL.md`（frontmatter + md）+ 選配；name＝資料夾名。**別自造格式。**（待建時定 frontmatter 欄位 name/description/why。）
- **兩種 scope（關鍵，2026-06-07 使用者點破）**：
  - **專案 skill（現在）**：源在 `knowledge/skills/`（跟專案 repo committed），裝到**這專案的** `.claude/skills`。clone 專案就帶著——**沒有「跨專案 sync」**。
  - **個人/泛化 skill（未來）**：跨專案畢業（B）→ 升到 `~/.claude/skills`（個人、跨專案）。**跨機器/跨專案 sync 只在這裡才有意義**——屬 [往外長](2026-06-05-往外長.md)，未建。
- **安裝 = AI 固化時直接做**（capture 程序版）：AI 本來就在寫檔 → 順手複製 SKILL.md 到工具 skill 位置（.claude/skills…）→ 立刻可用，**不逼用 CLI**（呼應「寄生 AI 讀寫檔」）。
- **CLI 批次安裝（installDomainSkills + registry skillsDir）一度建、又砍了（2026-06-07）**：純 no-op（無 domain skill）+ 專案 scope 用 AI 直接裝就夠；它的非重複價值（跨 checkout 重生 loadable 副本，像 npm install）只在 cross-checkout/泛化才需要。→ **YAGNI 砍**（同 health CLI 紀律），真要時 ~15 行 + 3 registry skillsDir 易重接。觸發：第一個 domain skill 出現 + 不 commit `.claude/skills` 的跨機器需求，或跨專案畢業。

## 進化策略：不自主，但靠執行回饋（vs GEPA / Hermes）
GEPA / Letta / Hermes 有**自主自我改進**的 skill（依結果自己重寫，閉環）。knowie 用 oracle 光譜切得更精準：
- skill 有**部分 oracle**（執行結果）→「**偵測 skill 出問題**」可**自動**（結果爛 → 自動標記待修；偵測可逆）。
- 「**重寫 skill**」**不可逆**、高 stakes → **人確認**。
- → **knowie 的 skill 進化 = 機器靠執行回饋察覺（自動）+ 人改寫（確認）**，不是 GEPA 閉環自主。這正是 [history/001](../history/001-從自動到人在環.md)（從自動到人在環）+ 延伸原則 5（可逆自動、不可逆要人）的 skill 版——autonomous skill self-edit 正是 Letta 自曝 risky 的坑，skill 版坑更深。
- trade-off（誠實）：GEPA 快（無人瓶頸）、knowie 慢但可審；賭注是「why + 人共識」差異化，不是進化速度。用速度換可控是一致選擇。

## 跨專案畢業：程序層可能領頭打進 `往外長`
domain skill 在多專案反覆出現 → 它是**方法論級**、該畢業到「個人小腦」（如概念向母概念收斂）。
- **基礎設施已存在**：agentskills.io / Claude 本就分 `~/.claude/skills`（個人,跨專案）vs `.claude/skills`（專案）→ 畢業 = 專案小腦 → 個人小腦（裝到個人位置）。knowie 只加 **why + 畢業判準**。
- **洞**：[往外長](2026-06-05-往外長.md) 的跨專案層「結構未定」，但**程序層最 tractable**——skill 離散、可攜、個人位置已存在；跨專案 why/概念模糊得多。→ **knowie 進跨專案層,從程序記憶下手**，skill 先畢業、why/概念隨後。

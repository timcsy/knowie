## 核心洞察
knowie 以前只做了**語意記憶**（三視角）。完整的記憶系統還有工作、情節、程序、前瞻、後設等器官——knowie 該長齊。

## 根本目的：可回憶（最上層）
knowie 的目的不是「管 why」，是**可回憶**——讓專案過去的認知能被完整重建（回到當初的「為何 / 怎麼 / 用了哪些概念」）。「按概念投影」「dispatch」「蒸餾」都是服務這個目的的手段。**這是新的根公理候選**（why → 可回憶）。

- **可回憶單元** = 一次認知的三面連結：為何（why）+ 怎麼（how）+ 用了哪些領域概念（concepts）。
- **張力**：dispatch 把一次認知分流到各類，會**碎片化**——回憶時拼不回「那一次」。
- **調和**：分流去各處（按類型存，利於「找所有原則」）+ 用「那一次」把碎片串成可回憶單元（利於「回到當初為何這樣」）。
- **episodes 的真角色 = 可回憶單元**：連結 why/how/概念、保留現場，對抗 dispatch 碎片化。不是「踩坑經歷」，是「回得去現場的那條線」。
- **兩種回憶**：語意檢索（找所有關於 X 的原則）+ 情節回憶（回到當初做 Y 為何那樣）。knowie 要同時支援。

## 記憶分類 → knowie 對應

| 記憶型態 | knowie | 軸 |
|---|---|---|
| 感覺記憶 | Context Window（輸入層，runtime，不儲存） | 現在 |
| 工作記憶 | `draft/` | 現在 |
| 語意記憶（陳述性） | `concepts/` | 過去／對世界 |
| 情節記憶（陳述性） | `episodes/`（新，尚未建） | 過去／對世界 |
| 程序記憶（非陳述性） | `skills/` | 能力 |
| 前瞻記憶 | vision 路線圖 / next | **未來** |
| 後設記憶 | Key Extensions / README 索引 / judge 健康檢查 | **對自己** |
| （meta）schema 轉換史 | `history/` | 系統自身演化 |

## 腦科學依據（併自 2026-06-05「記憶類比」）
為什麼這樣分層，有神經科學的趨同演化支撐：
- **趨同演化**：海馬迴（快寫快取）= `draft/`；新皮質（慢寫長期）= 三檔 + concepts；睡眠固化 = 批次蒸餾；突觸修剪 = 否決留墓碑。
- **為什麼非分層不可**：catastrophic interference（快寫直接改長期會撕裂舊記憶）→ 必須分兩層；固化是全局重組、不能線上做 → 整理必須離線批次。
- **但是「理想化記憶」**：可控、可審、可回滾；遺忘是主動且留痕（墓碑），不是被動消失。
- **分工**：AI 做 SWS（機械固化、修剪）、人做 REM（洞察、找母概念、定案）。

## 三視角的定位（關鍵）
- 三視角（principles/vision/experience）**不是某一層的內部**，而是**陳述性記憶（語意＋情節）結晶出的存取介面**。橫切，不限語意。
- 「**結晶**」調和了「衍生視圖 vs 人定權威」：它從底層記憶蒸餾**析出**，但析出後**自立、有權威**——不是每次重算的影子。
- **陳述性記憶 →（結晶）→ 三視角介面**；**非陳述性記憶（程序/skill）獨立，不結晶進三視角**（它是能力，不是知識內容）。

## episodes 的紀律
- 不存所有經歷：**篩選在 experience 入口就做了**（不值得記成經驗的，根本不進，也就沒 episode）。
- 結構：**一條 experience 教訓 ← 多個 episodes**。情節不合併（每次經歷獨立），語意才合併（多個情節蒸餾成一條教訓）。

## 兩個被補上的盲點
- **前瞻記憶**：其他記憶都在回顧過去，唯獨缺「未來」。vision 路線圖就是它（記得要往哪走）；next 是提取它。
- **後設記憶**：缺「對自己記憶的認知」。Key Extensions / judge 就是它（知道去哪找、健不健康）。

## 已收斂
- **資訊流方向：選 (i)**——三視角為主介面（寫和讀都在這）；記憶分類只是 backend 的「下沉歸類學」（語意→concepts、情節→episodes、程序→skills、schema→history）。三視角不降格。
- **不需立刻動結構**：現有 concepts/history/draft 已覆蓋；`episodes/` **按需出現**（等真有情節細節要下沉時才開），不預建。記憶框架是「下沉的歸類圖」，不是大重構。

## 程序記憶：自習得
讓 LLM 把常做的事自己蒸餾成 skill。

- **兩種來源**：meta skill（維護知識庫：capture/judge）+ **domain skill**（專案領域的重複工作：爬蟲建題庫、批量轉譯…）。domain 更有價值——幫專案長出「領域能力庫」。
- **= capture 的程序版**：察覺操作重複 → 記一條「候選 skill」進 draft（連這次怎麼做的）→ 反覆 / 成熟 → 固化成 skill。不需新引擎、不需操作日誌（靠在場察覺，把過眼即忘的 transcript 搶救成 draft）。
- **邊界**：做一次 → spec 工具實作；反覆做 → 固化成 domain skill（複習效應）。
- **守紀律**：skill 化是定案級、且影響更大（被執行，非被引用）→ AI 提議、人確認，比知識定案更慎。

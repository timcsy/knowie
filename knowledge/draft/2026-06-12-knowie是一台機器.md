# knowie 是一台機器：計算原語的趨同

> 2026-06-12 討論。draft，觀察型（meta）。
> 怎麼冒出來的：使用者發現「程式設計的概念一直在這裡長出來，Assembly 處理問題的方式（call stack 等）這裡也有」。這是**第三面同源鏡子**（前兩面：分散式系統、作業系統，見 [多agent併發](2026-06-12-多agent併發-git模型.md)）。

## 核心：knowie 結構上就是一台機器
計算原語一個個落出來，**不是巧合是趨同**。knowie 有一台機器該有的東西：
- **記憶階層**（context→draft→三檔→git）
- **calling convention**（原子邊界：prime/reconcile/consolidate）
- **call stack**（巢狀原子）
- **特權模式**（人 ＝ ring 0 / interrupt handler）
- **一致性模型**（git/CRDT/最終一致）

## 結構性對應（有承重機制的才收）

### call stack ＝ 巢狀 development atom
（接 [harness-loop 的 development atom](2026-06-12-harness-loop-workflow定位.md)：原子會巢狀——loop 生 helper、goal 拆 subgoal。）
| 機器 | knowie |
|---|---|
| call（push frame） | 父原子 spawn 子原子，給它 prime |
| stack frame（local、return 時丟） | 子原子的 **draft scratch**——做完就丟，只留 consolidate 出的耐久部分 |
| return value | 子原子 consolidate 往上傳的結果/教訓 |
| return address | 回父原子要續的地方（vision/next 的「接下來」） |
| heap / globals（跨 frame 持久） | **`knowledge/`**——所有原子共享的 why |
→ **「development atom」+「call stack」＝ knowie 定義了一套 calling convention（ABI）**：prime ＝ 傳參數進、consolidate ＝ 傳回傳值出、原子不透明 ＝ callee 內部自理。**原子邊界就是 knowie 的 ABI。**

### 記憶階層（最乾淨）
```
registers → cache → RAM → disk
context window → draft → 三檔(knowledge) → git/history
```
機器的記憶階層 ＝ [記憶系統](../concepts/記憶系統.md) 的快寫/慢寫梯度，同一個東西。

### 其餘真的對得上的
- **stack overflow** ＝ 原子無限巢狀不 return（goal 一直拆、helper 一直生、從不 consolidate）→ 真實失敗模式；解法同機器：**限制深度 / 每次 return 強制 consolidate**（像 tail-call）。
- **register spilling**（暫存器滿→寫 stack）＝ **context window 滿 → 寫進 draft**（draft ＝ spill space）。
- **interrupt → 特權 handler** ＝ **人在環/定案**——control 交給 ring 0（人/kernel）。接 [多agent併發](2026-06-12-多agent併發-git模型.md) 的保護環（延伸原則 5）。

## 為何趨同（你真正發現的）
**knowie 面對計算機從頭在解的同一個問題：「在快→慢的記憶階層上，協調有界的工作單元，加一個特權權威。」** CPU 這樣解（stack/heap/ABI）、OS 這樣解（process/protection）、分散式這樣解（consistency/ordering）。knowie 在**不同層級**（有界的**開發**工作、在 **why** 記憶階層上、加**人**權威）碰同一問題 → **落在同一組原語上**。

## 三面鏡子吻合 ＝ 真結構（非比喻）
分散式、OS、組合語言三面鏡子各自照、卻照出同一結構——這正是 [收斂](../concepts/收斂.md)「多條獨立推導指向同一個根 ＝ 可能是真的根」+ experience「外部框架是鏡子」。三面都吻合 → 強烈暗示 knowie 摸到真的底層結構。

## 守一條線：別硬湊每個 opcode（假統一）
陷阱 ＝ `收斂` 失敗模式的「**假統一**」。**判準：對應一個真的機制才收**（記憶階層、call stack、ABI、interrupt 都有承重機制 → 收）；硬掰「`MOV`/`XOR` 對應什麼」＝ 為統一而統一、壞抽象 → 別。

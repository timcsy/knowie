# draft —— 短期記憶（待沉澱的主題）

放從 chat 討論卸下來、但還不值得定案的東西。**一個主題塊一個檔**——一段聊出來、還沒定案的想法，連它的脈絡一起（不拆成零碎便利貼）。

## 命名：`YYYY-MM-DD-主題.md`
- 日期 = 那次討論：回想的時間線索，也標記這是一次討論的 episode 快照。
- 主題 = 語義線索。
- 兩個都在檔名，不必打開就能找（按日期排「上個月那批」、按主題找「關於 X 的」）。
- 同一主題下次再聊 → **開新檔（新日期）**，不覆蓋舊的；同主題累積出多個快照 = 「該固化」的信號。

## 運作模型（貼合短期 / 情節記憶）
- 記憶的單位是「一段（episode）」，不是原子——一檔一個主題塊。
- **預設衰減**：沒被再碰的主題預設淡出。遺忘是功能，大部分本來就該淡出，只有少數固化。
- **複習救起**：被反覆回來討論的（同主題多個快照），證明該固化 → 蒸餾進三檔 / concepts 後刪掉。
- **新近度看 git**：整檔 mtime = 這主題多久沒碰，不手寫。
- **三種去向都是離開 draft**：固化、淡出（刪）、否決（→ `history/` 墓碑）。

## judge 巡檢
用 git 新近度 找最久沒碰的主題塊，逐一判「救起（固化）or 放手（淡出）」。判準是 新近度 + frequency，不是年齡。

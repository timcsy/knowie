# 讓認錯變便宜（Make Admitting Error Cheap）

> 母概念。knowie 對抗「自我服務的漂移」的總機制。
> 回答：已定案、已據以開發的東西被發現**根本錯**（認知失調）時怎麼辦。

最危險的不是想法漸進過時（superseded），是「**已定案 + 已行動的東西被發現根本錯**」——沉沒成本 + 確認偏誤誘人**合理化維持**，而非認錯。死撐的最大藉口是「認錯成本太高」。knowie 的解：**逐項把認錯成本壓到低於合理化的誘惑**。

## 三視角投影
- **規範**：認錯便宜是設計義務——每個機制都該**降低**認錯成本，不該升高。
- **處境**（發現路線錯時的流程）：承認 + 標 🔴 不立刻刪 → 解剖**是哪個根前提錯**（不是整條錯，把大危機縮成「一個根的問題」）→ 修訂（若錯的是根，走根本原則修訂特殊路徑）→ 沿推導鏈**連帶更新** → 蒸餾「為何會選錯」成 experience 教訓。
- **存在**：最深的敵人是自我服務的漂移 / 合理化維持；**knowie 會不會爛，很大程度取決於能不能讓人 + AI 願意認「已定案的錯」**。

## 降低認錯成本的機制（都已實作）
- **因果軌跡 + superseded 不刪**：認錯 = 記一條轉移，不是摧毀；錯誤留著當最寶貴的 why（「為何不能走這條」）。
- **可回憶單元**（episodes）：回得到「當初為何選這條」，精確診斷哪個根錯，不必整團丟棄。
- **推導鏈 + judge blast-radius**：錯根牽連哪些衍生，可機械列出、連帶更新（不漏）。
- **根本原則修訂特殊路徑**：現實反饋有資格挑戰原則——[history/004](../history/004-補使命-根公理二鬆綁.md) 就是一次實例。

## 限制（誠實）
knowie **不替你發現方向錯**——方向性的錯要靠人對現實的洞察；knowie 只偵測局部 🔴 不一致。它給的是「認錯後**有序** + 認錯**便宜**」，不是「自動發現錯」。

# 記憶系統（knowie as a Memory System）

> 母概念。knowie 對應一整套認知記憶分類；三視角是其上的**結晶介面**。

knowie 不只是「語意記憶」（三視角）——它對應一整套認知科學的記憶器官。把知識往對的記憶層下沉，是整理的歸類學。

## 記憶型態 → knowie 對應
| 記憶型態 | knowie | 軸 |
|---|---|---|
| 感覺記憶 | context window（runtime，不儲存） | 現在 |
| 工作記憶 | `draft/` | 現在 |
| 語意記憶（陳述性） | `concepts/` | 過去／對世界 |
| 情節記憶（陳述性） | `episodes/` | 過去／對世界 |
| 程序記憶（非陳述性） | `skills/` | 能力 |
| 前瞻記憶 | `vision` 路線圖 / next | 未來 |
| 後設記憶 | Key Extensions／索引／judge | 對自己 |
| schema 轉換史 | `history/` | 系統自身演化 |

## 關鍵：三視角是「結晶介面」，不是某一層
- 三視角（principles/vision/experience）橫切**陳述性記憶**（語意＋情節），是它**結晶**出的存取介面——不是某個記憶層的內部。
- 「結晶」調和「衍生視圖 vs 人定權威」：從底層蒸餾析出，但析出後**自立、有權威**，不是每次重算的影子。
- 資訊流：**讀寫都在三視角介面**；記憶型態只是 backend 的「下沉歸類學」（語意→concepts、情節→episodes、程序→skills、schema→history）。**三視角不降格**。
- 非陳述性記憶（程序/skill）**獨立，不結晶進三視角**（它是能力，不是知識內容）。

## 可回憶單元：對抗 dispatch 的碎片化
- [分發非傾倒](分發非傾倒.md) 把一次認知分流到各層，利於「找所有關於 X 的原則」（語意檢索），但會**碎片化**——回憶時拼不回「那一次」。
- 解：**episodes = 可回憶單元**（why + how + 用了哪些概念，串回現場），對抗碎片化。兩種回憶並存：語意檢索 + 情節回憶。

## 三視角投影
- **規範**：detail 往對應記憶層下沉（按歸類學），不堆在三檔；三視角是介面。
- **處境**：知識長大時下沉到對的層；`episodes/` **按需開**（有可回憶單元要存才開），不預建——記憶框架是「下沉的歸類圖」，不是大重構。
- **存在**：knowie 是**理想化記憶**——可控、可審、可回滾；遺忘主動留痕（墓碑／superseded），不是被動消失。

## 腦科學依據（為何非分層不可）
趨同演化撐這個分層：海馬迴（快寫）=`draft/`、新皮質（慢寫）=三檔+concepts、睡眠固化=批次[蒸餾](蒸餾.md)、突觸修剪=否決留墓碑。**catastrophic interference**（快寫直接改長期會撕裂舊記憶）→ 必須分兩層；固化是全局重組、不能線上做 → 整理必須離線批次。分工：AI 做機械固化／修剪，人做洞察／定案。

## backend 可插拔
大規模召回／檢索**不是 knowie 自建**：why 結構與代謝（capture/next/judge）留純 markdown，大規模 recall 外掛後端（向量／FTS5／LLM Wiki），檔案＝真相源、工具＝衍生索引；程序層整合 agentskills.io（別自造）。決策見 [history/003](../history/003-連結慣例與檢索定位.md)；FUSE 掛載未證仍在 draft。

# 意義可追溯改寫（Meaning Is Retroactively Mutable）

> 母概念。knowie 為何存在的最深一層；也是 [記憶系統](記憶系統.md) god-concept watch 預測會自立的那塊（event-sourcing / replay / 因果 log 在此落腳）。

## 一句話
**事實寫一次、向前寫；意義讀很多次、向後讀。** 過去不會變，但**過去的意義會被未來改寫**——所以你不能把意義凍進當下的快照，只能保留不可變的記錄，讓意義隨未來重新投影。

## 兩個半邊的張力
- **發生**：向前、因果、一次性、不可逆（不能 un-happen 一個事件）。
- **理解**：向後、全局、**永遠在改**。

衝突點：一旦把「此刻怎麼理解」壓成 snapshot 並丟掉事實，就被那一刻的理解綁架了——未來想用新角度重新理解，**沒有原料**。

## 核心斷言：事件不可變，意義可被追溯地改寫
過去那天就定了，但「它是什麼意思」是**一個在固定記錄上移動的標的**，而且常是被**未來**移動的：
- 一個決策只有在後來被推翻時，才回溯獲得「轉移」的意義（見 [history](../history/) 的本質）。
- 一個當下合理的設計，環境一變才回溯變成破綻（battle：M1「整卡 button」→ feature 007 才塌）。
- 一個被否決的選項，環境變了墓碑會復活。

所以**不能把意義烤進 snapshot**——因為意義還沒定、也永遠不會 final。

## 推論：留 log、按需投影，而非凍結 snapshot
> **保留 log 不是為了回答你已想到的問題，是為了能回答你還沒想到的問題。**
> log = 對開放未來的 query optionality；snapshot = 對固定問題的預先答案。

這就是 **snapshot → event-sourcing**、**RNN hidden-state → Transformer KV-cache**、**migrate snapshot-判斷 → git-replay**（[history/009](../history/009-migrate運作模型從snapshot判斷到時間軸replay.md)）**同一個 move 三次**的根：別把歷史壓成有損當前態；留不可變 log，需要時再 fold / attend 出意義。對應地——**因果**（投影 as-of-t 只能用 ≤t 的事件）是這條的時間紀律；**多投影/CQRS/三視角**是同一 log 養多個 read model。

## knowie 的特例
- **code = WHAT 的 snapshot**（當下凍結）；**why = 那個可追溯改寫的意義**（永不 final、讀不出來自當前 code）。
- knowie 存在的唯一理由就是這條的特例：**你無法從凍結的現況讀出演化中的意義，所以必須保留 why 的 log。** git 留 WHAT 的事件 log；knowie 留 **WHY 的投影 log**（`history/` = 意義轉移的軌跡）。
- 與 [why沒有oracle](why沒有oracle.md) 是 why 的兩個性質：why **無 oracle**（驗不了）+ why **可追溯改寫**（不會定）。migrate 難，正因它對「無 oracle 又會改寫的意義」、從「不為它而記的 log」反推。

## 為什麼這麼多面鏡子都指它
不是某領域的技巧，是**任何要「在時間裡保持理解」的系統都逃不掉的結構**：DB／CPU／神經網路／史學／宗教／git——只要**既要忠於發生過什麼、又要容許理解持續演化**，就會長出「不可變記錄 + 因果 + 按需投影」這台機器（見 [收斂](收斂.md)：多條獨立推導指同一根 → 可能是真的根）。人的版本：**聖經一讀再讀**——文本固定、讀法無窮；留住固定文本，正因知道此刻的理解不是最終的。

## 樸素版
**我們不願讓「此刻怎麼理解」奪走「以後能不能重新理解」的權利。** knowie 把這句話套在「一個專案的為什麼」上。
